# Loading / Empty State Philosophy

**Purpose:** Keep state UX emotionally comfortable and operationally clear.

**Who uses it:** AI + human devs.

**Change frequency:** Moderate.

## Principles
- Loading should feel neutral and breathable.
- Empty should feel calm guidance, not error-like.
- Avoid “empty flash” if data fetching is async and may delay.

## What belongs here
- State tone language ladder
- Guidance on skeleton vs empty copy

## What not to include
- Feature-specific copywriting
- Business logic / provider implementation
