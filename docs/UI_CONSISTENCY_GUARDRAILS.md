# UI Consistency Guardrails (NexaLedger)

**Purpose (CRITICAL):** Prevent UI drift and preserve premium operational identity.

**Who uses it:** AI and human devs.

**Change frequency:** Low–Moderate (small updates only).

## Guardrails (summary)
- Border radius: consistent surface/control rounding.
- Shadow softness: avoid heavy shadows on lightweight surfaces.
- Hover lift: restrained; match existing motion tokens.
- Transition timing: prefer `duration-200/300 ease-out` family.
- Spacing rhythm: preserve breathing room scales.
- Typography hierarchy: keep semantic tiers consistent.
- Opacity: limit opacity variants per tier.
- Card density: avoid overcrowding/competing visual anchors.
- Button behavior: focus-visible parity always.
- Empty/loading tone: calm guidance ladder.
- RTL positioning: fixed/floating actions must follow RTL convention.
- Mobile spacing: avoid density spikes.
